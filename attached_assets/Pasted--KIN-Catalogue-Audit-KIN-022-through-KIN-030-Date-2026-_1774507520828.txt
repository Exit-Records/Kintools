## Per-Tool Details

### KIN-022 Fair Share (Medium risk — stores group expenses via IndexedDB)

**Storage:** IndexedDB primary (FairShareDB), localStorage fallback (fairshare_state)

PASS:

- Single-file HTML, Blob manifest, Blob SW (kin022-v1)
- Full expense splitting: equal, unequal, percentage, shares
- Debt simplification algorithm (greedy creditor/debtor matching) correct
- Balance calculation correct (payer gets +amount, participants get -share)
- JSON export/import with duplicate detection and merge
- Text summary export to clipboard
- 10 expense categories
- Tip calculator with presets and custom %
- Percentage calculator (3 modes: X% of Y, X is ?% of Y, % change)
- XSS prevention via esc() helper using textContent→innerHTML
- kinConfirm() for destructive actions
- Local date formatting correct
- ⓘ storage popover present

ISSUES:

- [MEDIUM] Dark mode uses `[data-theme="dark"]` CSS variable pattern, not body.dark with !important. Memory says three patterns are in use and to preserve whichever the source uses, but this is a fourth pattern not documented.
- [LOW] Theme key `fs-theme` not tool-specific format (should be `kin022-theme`)
- [LOW] Theme-color meta has no `id` attribute (should be `id="theme-color-meta"`)
- [LOW] Bug report: old pattern (Content-Type text/plain, sends email field)
- [LOW] No OG/canonical tags
- [LOW] External Google Fonts (DM Sans + DM Mono) not cached by SW

### KIN-023 JSON Formatter (Low risk, stateless)

**Storage:** Theme only in kin-json-theme

PASS:

- Single-file HTML, Blob manifest, Blob SW (kin-json-v2)
- 4 modes: Format, Tree, TypeScript interface generation, JSON diff
- Auto-fix for malformed JSON (trailing commas, single quotes, unquoted keys)
- Error position reporting (line/col) from parse errors
- JSON path query with dot/bracket notation
- Syntax highlighting with type-specific colours
- Collapsible tree view with item counts
- Stats bar (bytes, keys, depth, objects, arrays)
- Sort keys deep (recursive)
- Minify function
- Tab key inserts spaces in textarea
- Cursor position preservation across re-renders (saveCursor/restoreCursors)
- XSS: esc() function present
- Diff with added/removed/changed categorisation

ISSUES:

- [LOW] SW cache key `kin-json-v2` not standard format (should be `kin023-v1` or similar)
- [LOW] Theme-color meta id is `meta-theme` not `theme-color-meta`
- [LOW] @import for Google Fonts placed after :root declaration (must be first in style block per build rules)
- [LOW] Manifest icons reference `icon.svg` which doesn’t exist
- [LOW] Bug report: old pattern (Content-Type text/plain, sends email)
- [LOW] No OG/canonical tags
- [LOW] No ⓘ popover (correct — stateless)

### KIN-027 Kin Gym (High complexity — stores workout data, exercises, routines, log)

**Storage:** kin-gym-theme, kin-gym-ex, kin-gym-eid, kin-gym-rt, kin-gym-log

PASS:

- Single-file HTML, Blob manifest, Blob SW (kingym-v3)
- 4 tabs: Rest timer, Interval timer, Workout builder, Log
- Rest timer with SVG ring progress, presets (30s-3min), custom duration
- Interval timer with configurable work/rest/rounds
- Workout builder with 4 exercise types (strength, bodyweight, duration, cardio)
- Set tracking with warm-up/failure/drop-set tags
- Auto rest timer trigger on set completion
- Session timer with set counter
- Previous workout data display per exercise
- Routine save/load/rename/delete/share system
- Workout sharing via base64-encoded URL parameters
- WAV audio generation engine (22kHz, 16-bit PCM) for iOS-compatible sound
- Audio pool with 4 pre-created Audio elements for concurrent playback
- iOS audio unlock on first user gesture
- Custom modal system (modalConfirm/modalPrompt) instead of kinConfirm
- Wake lock support
- Demo mode with sample exercises and log
- `body.dark` with extensive !important coverage
- Theme key kin-gym-theme (standard format)
- Theme-color meta id=“theme-color-meta” ✓
- Bug report uses mode:‘no-cors’ ✓ (first in this batch to do so)
- By William and Darren (correct credit)
- ⓘ storage popover present

ISSUES:

- [MEDIUM] Custom modal system instead of kinConfirm() — works well but diverges from standard
- [LOW] esc() only escapes `"` and `<` but not `>` and `&` — incomplete for general XSS prevention (exercise names from user input)
- [LOW] No OG/canonical tags
- [LOW] Bug report submit has no email/name field at all (no anonymous toggle). Correct per ethos but differs from other tools’ bug report format.
