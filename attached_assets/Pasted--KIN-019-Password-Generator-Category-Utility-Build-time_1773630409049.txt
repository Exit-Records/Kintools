# KIN-019 - Password Generator

**Category:** Utility
**Build time:** 2-3 hours
**Audience:** Everyone, especially developers
**Replaces:** Ad-heavy password generator sites
**Why now:** Universally needed, trivial to build, immediately credible. Developers who find this will explore the rest of the catalog.

-----

## Core Concept

A fast, private, zero-storage password generator that runs entirely in the browser. No accounts, no tracking, no ads. Generate strong passwords and passphrases with full control over format and character composition.

-----

## Trust Statement

Visible in the UI at all times, not hidden behind a link:

> Nothing leaves your browser. No storage. No tracking. No server calls.

This is the primary differentiator from ad-heavy alternatives. Make it prominent, not buried.

-----

## Features

### 1. Random Password Generation

The default mode. User sets parameters, hits generate, gets a password.

- **Length slider:** Range 8-128 characters, default 20
- **Character type toggles:**
  - Uppercase (A-Z)
  - Lowercase (a-z)
  - Numbers (0-9)
  - Symbols (!@#$%^&* etc.)
- **At least one toggle must remain active** (prevent empty generation)
- **Copy button** with visual confirmation (brief ‚ÄúCopied‚Äù state)
- **Regenerate** without changing settings

### 2. Passphrase Mode

Toggle between character-based passwords and word-based passphrases.

- **Word count:** 3-8 words, default 4
- **Separator character:** Configurable (hyphen, space, period, underscore, number, custom)
- **Capitalisation toggle:** Title case, all caps, all lower, random per word
- **Optional number/symbol injection:** Append or insert a digit or symbol to satisfy services that require mixed character types
- **Dictionary:** Use a curated wordlist (e.g. EFF long wordlist, ~7,776 words). Bundle it in the app, no external calls.

### 3. Strength Indicator

Visual feedback that updates in real time as the user adjusts settings.

- **Entropy calculation:** Show bits of entropy based on character pool size and length
- **Time-to-crack estimate:** Display human-readable estimate (e.g. ‚Äú~3 centuries at 1 billion guesses/second‚Äù)
- **Visual bar or gradient:** Colour shift from red through amber to green
- **No vague labels** like ‚ÄúStrong‚Äù or ‚ÄúWeak‚Äù without the numbers backing them up. Show both the label and the math.

### 4. Exclude Ambiguous Characters

Toggle (off by default) to remove visually similar characters:

- `0` and `O`
- `1`, `l`, and `I`
- `|` and `l`
- `5` and `S` (optional, configurable)

Useful when passwords need to be read aloud, printed, or typed from a screen.

### 5. Custom Character Sets

Expandable/advanced section for users who need fine control:

- **Include only:** User defines exact allowed characters
- **Exclude specific:** User specifies characters to remove from the default set
- Addresses services with unusual restrictions (e.g. ‚Äúno special characters‚Äù, ‚Äúonly these symbols: !@#‚Äù)

### 6. Pronounceability Toggle

Alternative generation mode for passwords that are easier to type or say aloud.

- Alternating consonant-vowel patterns (e.g. `bokami-tuvelo`)
- Configurable syllable count
- Optional digit/symbol suffix for compliance
- Note in UI: ‚ÄúLower entropy per character. Compensate with length.‚Äù

### 7. Batch Generation

Generate multiple passwords at once.

- **Count selector:** 1-20 passwords
- **List view** with individual copy buttons per row
- **Copy all** button (newline-separated)
- All passwords use the current settings

### 8. Session History

Show previously generated passwords during the current browser session.

- **Visible list** below the generator, most recent first
- **Individual copy buttons** per entry
- **Clear history** button
- **Auto-cleared on tab/window close** (no localStorage, no persistence)
- **Visual label:** ‚ÄúSession only. Cleared when you close this tab.‚Äù

### 9. Keyboard Shortcuts

For power users who generate passwords frequently.

- `Space` - Regenerate
- `Enter` or `Cmd/Ctrl+C` (when focused) - Copy to clipboard
- `Tab` - Cycle through settings
- Display shortcut hints in the UI (subtle, not cluttered)

## Technical Notes

### Architecture

- **Single-page app.** No backend, no API calls, no external dependencies at runtime.
- **Crypto:** Use `crypto.getRandomValues()` for all randomness. No `Math.random()`.
- **Bundled wordlist:** EFF long wordlist embedded in the app. No fetch calls.
- **Framework:** React (single .jsx file for KIN deployment) or vanilla HTML/JS depending on KIN stack conventions.

### What This App Does NOT Do

- Store anything (no localStorage, no cookies, no IndexedDB)
- Make network requests
- Track usage or analytics
- Require an account
- Show ads

### Accessibility

- All controls keyboard-navigable
- ARIA labels on toggles and sliders
- High contrast support
- Screen reader-friendly copy confirmations

### Responsive

- Works on mobile, tablet, desktop
- Touch-friendly toggle sizes
- Slider works with touch input

## UI/UX Direction

- **Clean, minimal.** No decoration for its own sake.
- **Dark mode default** with light mode toggle (or respect system preference)
- **Monospace font** for generated passwords
- **Single scroll.** Everything visible without navigation.
- **Advanced options** (custom characters, pronounceability) collapsed by default, expandable
- **Trust statement** always visible, not in a footer
